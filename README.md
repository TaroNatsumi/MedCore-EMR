# MedCore - Децентрализованная Медицинская Информационная Система

MedCore — это инновационная платформа электронных медицинских карт (ЭМК), объединяющая клиники, врачей и пациентов в единую безопасную сеть. Проект разработан с фокусом на криптографическую защиту данных, маршрутизацию баз данных и строгое разделение прав доступа (RBAC).

## 🌟 Ключевые особенности

- **Шифрование медицинских данных (AES-256):** Все диагнозы, жалобы и назначения пациентов шифруются перед сохранением в базу данных. Доступ к расшифровке возможен только по уникальному токену и ПИНФЛ пациента.
- **Многоуровневая архитектура БД:** Проект использует несколько изолированных баз данных (`db.sqlite3`, `employees.sqlite3`, `temp.sqlite3`) с автоматическим роутингом запросов для повышения безопасности и распределения нагрузки.
- **Ролевая модель доступа (RBAC):**
  - **MedCore Control (Суперадмин):** Глобальное управление филиалами больниц.
  - **Администратор клиники:** Управление персоналом и аудит безопасности (трекинг доступа к картам).
  - **Врач:** Просмотр истории болезни по ПИНФЛ, добавление новых зашифрованных записей, выписка направлений.
  - **Регистратура:** Регистрация пациентов, генерация криптографических ключей.
  - **Пациент:** Личный кабинет ("Зеркало Здоровья") с доступом к своей карте и возможностью экспорта в PDF.
- **Современный UI/UX:** Адаптивный дизайн, реализованный на Tailwind CSS (Glassmorphism, плавная анимация).

## 🛠 Технологический стек

- **Backend:** Python 3, Django
- **Frontend:** HTML5, Tailwind CSS, JavaScript (Vanilla)
- **База данных:** SQLite (с кастомной маршрутизацией)
- **Безопасность:** Интегрированные алгоритмы шифрования (AES-256)
- **Экспорт:** Генерация PDF-отчетов (ReportLab)

## 🚀 Установка и запуск (Локально)

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/yourusername/medcore.git
   cd medcore
   ```

2. **Создайте и активируйте виртуальное окружение:**
   ```bash
   python -m venv venv
   # Для Windows:
   venv\Scripts\activate
   # Для Linux/Mac:
   source venv/bin/activate
   ```

3. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```
   *(Если файла `requirements.txt` еще нет, установите: `pip install django reportlab cryptography pycryptodome`)*

4. **Примените миграции (базы данных создадутся автоматически):**
   ```bash
   python manage.py migrate
   python manage.py migrate --database=employees_db
   python manage.py migrate --database=temp_db
   ```

5. **Запустите локальный сервер:**
   ```bash
   python manage.py runserver
   ```
   Откройте бразуер и перейдите по адресу: `http://127.0.0.1:8000/`

## 🌍 Подготовка к деплою (Развертывание на сервере)

При публикации проекта на бесплатном сервере (например, PythonAnywhere, Render, Railway), обратите внимание на следующие настройки в `hospital_system/settings.py`:

1. **Безопасность:**
   - Замените `SECRET_KEY` на новый, сложный ключ и не светите его в публичном репозитории.
   - Установите `DEBUG = False`.
2. **Хосты:**
   - Добавьте домен вашего сервера в `ALLOWED_HOSTS = ['yourdomain.com', 'localhost']`.
3. **Статические файлы:**
   - Настройте сбор статики (`STATIC_ROOT`) и выполните команду `python manage.py collectstatic` перед запуском на сервере.
